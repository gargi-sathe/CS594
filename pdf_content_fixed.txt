Idea 
The central idea of this 
project is to move 
beyond optimizing 
individual delivery routes 
and instead design the 
entire delivery system so 
that ultra-fast delivery 
(10–15 minutes) becomes 
achievable by 
construction.
The Problem
The core problem is not 
routing; it is system 
design: where to place 
warehouses, how to 
position drivers, and 
how to structure the 
network so fast delivery 
is even possible.
Significance
Capital efficiency in 
ultra-fast logistics relies 
strictly on Service Level 
Agreement (SLA) 
adherence without 
unbounded driver 
subsidies. System design 
must account for 
operational variance.
Hypothesis
Even with sufficient total 
drivers (D), poor spatial 
distribution of drivers and 
warehouses leads to 
localized shortages, which 
significantly increases 
delivery times.
A system that dynamically 
aligns driver positioning 
and warehouse placement 
with demand hotspots will 
reduce delays and improve 
10–15 minute delivery 
success rates.

Evaluation Plan
Key Metrics Stress Test Scenarios Baselines We Compare 
Against
● On-time delivery rate (% 
of orders within 10–15 min 
target)
● Tail latency (p90 and p95 
delivery times)
● Cost per order 
(warehouse + driver cost)
● Orders missed or rejected
● Demand spike in one 
region
● Driver shortage (fewer 
drivers available)
● Warehouse closure (one 
hub goes offline)
● Time-of-day demand 
shifts (morning vs evening 
patterns)
● Demand-only clustering 
for warehouse placement
● Coverage-based heuristic 
placement
● Nearest-warehouse 
Voronoi for sectoring
● Nearest feasible driver for 
dispatch

Approach
Overall Approach
We model the system as a hierarchical 
delivery optimization problem, with 
decisions at three levels:
● Strategic: Warehouse placement 
(learned using RL)
● Tactical: Driver sectoring and 
staging (heuristics → RL extension)
● Operational: Dispatching and 
routing (heuristics → RL extension)
This structure ensures a working system 
early, while enabling meaningful research 
contributions.
Proposed Scope
The core project focuses on building and 
evaluating a simulation-based delivery 
system:
● Develop a city-scale simulator 
(map, demand, drivers, 
warehouses)
● Implement a complete heuristic 
pipeline for sectoring, dispatching, 
and routing
● Introduce RL for warehouse 
placement
● Evaluate performance using:
○ On-time delivery rate 
(10–15 min target)
○ Tail latency (p90/p95 
delivery times)
○ Robustness under 
demand and supply 
variations
Next Steps
1. Build simulator and baseline pipeline
2. Define evaluation metrics and test 
scenarios
3. Train RL agent for warehouse 
placement
4. Compare against heuristic baselines
5. Extend to more advanced 
components if time allows
End Result
● Input: city map + demand + K 
warehouses + D drivers
● Output: warehouse locations + 
delivery policy
● Goal: maximize % of orders delivered 
within 10–15 min
